# ТЗ: Биллинг клубов (B2B-подписки на платформу)

> Статус: черновик на согласование. Дата: 2026-06-05.

## 0. Важно: два разных биллинга — не путать
- **B2B (этот ТЗ):** платформа ↔ КЛУБ. Клуб платит за пользование платформой (SaaS-подписка).
- **B2C (уже реализовано):** клуб ↔ его КЛИЕНТЫ (депозиты, тарифы на время, постоплата, магазин).

Этот документ — только про B2B.

## 1. Цель и модель
Платформа зарабатывает на клубах. Поток:
1. У клуба есть **кошелёк (баланс)** в платформе.
2. **Суперадмин** пополняет баланс клуба вручную через `/platform` (пока без эквайринга).
3. **Владелец клуба** покупает/продлевает **тариф** (Free / Starter / Business) — списывается с баланса.
4. Пока подписка активна → клуб пользуется платформой. Истекла/нет денег → доступ ограничивается.

## 2. Роли
| Роль | Где | Что может |
|---|---|---|
| Суперадмин | `/platform` | пополнять баланс клуба, видеть балансы/подписки/историю, вручную выдавать/продлевать подписку, корректировки |
| Владелец клуба | клубная админка | видеть баланс и историю, покупать/продлевать тариф с баланса, видеть срок подписки |

## 3. Тарифы (уже есть — `SubscriptionPlan`)
| Тариф | Цена/мес | Лимит ПК |
|---|---|---|
| Free | 0 | до 5 |
| Starter | X сум | до 20 |
| Business | Y сум | без лимита |

Поля плана: `tier`, `name`, `monthly_price`, `max_pcs` (0 = ∞), `features`. **Модель списания — фикс за месяц** (`monthly_price`). Free бесплатен, но с лимитом ПК.

## 4. Модель данных

### Новое
- **ClubWallet** (1:1 с Club)
  - `balance` Decimal(12,2), default 0 — не уходит в минус
  - `updated_at`
- **ClubWalletTransaction** (леджер всех движений)
  - `wallet` FK
  - `type`: `topup` (пополнение суперадмином) / `charge` (списание за подписку) / `refund` / `adjust` (ручная корректировка)
  - `amount` Decimal (знак по типу)
  - `balance_after` Decimal (снимок баланса после операции)
  - `comment`, `created_by` (кто провёл), `created_at`

### Существующее (используем как есть)
- **ClubSubscription**: `plan`, `status` (trial/active/promised/expired/blocked), `expires_at`, `auto_renew`.
- **Club.trial_until**, задача `enforce_subscriptions` (гасит триалы, блокирует должников).

## 5. Бизнес-правила
1. **Пополнение:** суперадмин кредитует баланс → транзакция `topup` (+amount). Только суперадмин.
2. **Покупка/продление тарифа** (владелец):
   - если `balance >= plan.monthly_price` → списать (`charge`), `subscription.plan = plan`, `status = active`,
     `expires_at = max(now, текущий expires_at) + 30 дней` (продление не сгорает).
   - иначе → 400 «Недостаточно средств, пополните баланс».
3. **Free:** 0 сум, доступен всегда без баланса, лимит 5 ПК.
4. **Авто-продление** (`auto_renew`, опц.): за N дней до `expires_at` пытаемся списать `monthly_price`; хватает → продлить; не хватает → уведомление, по истечении → `expired` → блок.
5. **Лимит ПК:** при активном плане нельзя добавить ПК сверх `max_pcs` (создание ПК → проверка).
6. **Гейт доступа:** статус не `active`/`trial` → ограничение (режим — см. §7).
7. **Баланс не уходит в минус**; все операции атомарны и пишут транзакцию.

## 6. API

### `/platform` (суперадмин)
- `GET  /api/v1/platform/clubs/` — клубы: баланс, план, статус, `expires_at`, кол-во ПК.
- `GET  /api/v1/clubs/<id>/wallet/` — баланс + история транзакций.
- `POST /api/v1/clubs/<id>/wallet/topup/` `{amount, comment}` — пополнить.
- `POST /api/v1/clubs/<id>/subscription/grant/` `{plan, days}` — выдать/продлить **вручную** (без баланса, например бонус/тест).

### Клубная админка (владелец)
- `GET  /api/v1/subscription-plans/` — список тарифов.
- `GET  /api/v1/clubs/<id>/subscription/` — текущий план/статус/срок (частично есть).
- `GET  /api/v1/clubs/<id>/wallet/` — баланс + история (списания владельцу видны).
- `POST /api/v1/clubs/<id>/subscription/buy/` `{plan}` — купить/продлить с баланса.

Все эндпоинты с проверкой прав (суперадмин / владелец своего клуба).

## 7. Гейт доступа (enforcement) — РЕШЕНИЕ НУЖНО
Permission/middleware: для запросов клубной админки (и опц. шелла) проверяет статус подписки.
- Не `active`/`trial` → ответ `402/403` со спец-кодом → фронт показывает экран **«Подписка неактивна → оплатите»**.
- `enforce_subscriptions` (есть) выставляет статусы + добавим **авто-списание** при auto_renew.

**Вариант блокировки (выбрать):**
- **A. Жёсткая** — админка + шелл блокируются (только экран оплаты). Максимально стимулирует оплату.
- **B. Мягкая (грейс N дней)** — несколько дней баннер-предупреждение, потом блок.
- **C. Только админка** — управление блокируется, активные клиентские сессии на ПК доигрывают.

## 8. UI
**`/platform` (карточка клуба):** баланс, кнопка «Пополнить» (сумма + комментарий), текущий план + срок, «Выдать подписку», лента транзакций, индикатор «скоро истекает».

**Клубная админка → раздел «Подписка»:** баланс, текущий план + дата окончания, карточки тарифов с кнопкой «Купить/Продлить», история списаний, предупреждение если < N дней до конца, тумблер авто-продления.

## 9. Краевые случаи
- Недостаточно баланса при покупке/авто-продлении → не продлеваем, по сроку → `expired`.
- Понижение плана при превышении лимита ПК → запретить, пока ПК > лимита нового плана (или предупредить).
- Смена плана в середине периода → MVP: новый период 30 дней от покупки (без пропорции).
- Возврат/корректировка — только суперадмин, транзакция `refund`/`adjust`.
- Параллельные списания → транзакция в БД + `select_for_update` на кошельке.

## 10. Этапы внедрения
- **MVP-1:** ClubWallet + транзакции + пополнение суперадмином (`/platform`) + покупка плана с баланса (владелец) + базовый гейт (вариант блокировки по выбору) + раздел «Подписка» в админке.
- **MVP-2:** авто-продление, грейс-период, уведомления (Telegram/в шелл), enforcement лимита ПК.
- **MVP-3:** отчёты в `/platform` — реальный MRR из `charge`-транзакций, ARPU клубов, прогноз.

## 11. Вне scope
- Реальный эквайринг (карты/СБП) — пока только ручное пополнение суперадмином.
- B2C-биллинг клиентов (уже реализован отдельно).

## 12. Открытые решения (подтвердить перед стартом)
1. **Модель списания:** фикс/мес `monthly_price` *(по умолчанию, как сейчас)* / предоплата днями / пер-ПК.
2. **Блокировка:** A жёсткая / B грейс / C только админка.
3. **Период:** 30 дней *(по умолчанию)*.
4. **Авто-продление:** вкл по умолчанию или вручную.
