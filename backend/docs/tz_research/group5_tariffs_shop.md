# Группа 5: Тарифы, Магазин, Настройки

Источник: 7 статей справки SmartShell (Тарифы, Магазин, Товары/услуги/комбо, Подакцизный товар, Штрихкод, Честный знак, Настройки).

## 1. Тарифы (4 типа)

Раздел: Доп. меню → Товары и услуги → Тарифы. Карточка содержит: название, тип, состояние, цену по залам, длительность, график действия, срок жизни. Информационные значки: полумесяц (пересекает полночь), процент (применимы скидки), глаз (виден в шелле), монетка (виден оператору в продаже), звезды (выделен в шелле), красный календарь (праздничные дни), бумажный самолётик (Telegram-оповещение о продаже), синий календарь (онлайн-бронь).

### 1.1 FIXED — Фиксированный
Действует ровно указанную длительность с момента покупки (например, 3 часа). Деньги списываются сразу полностью. Длительность — обязательное поле. График продажи опционален.

### 1.2 PACKAGE — Пакетный
Длительность не задаётся — определяется графиком действия. Пример: график действия 22:00–08:00 → купив в 22:00 получаешь 10 ч, купив в 05:00 — 3 ч. Время продажи можно ограничить отдельно. Заканчивает работу с окончанием времени продажи/действия. При досрочном выходе остаток сохраняется до окончания тарифа.

### 1.3 PER_MINUTE — Поминутный (Starter/Business)
Длительность фиксируется в 1 минуту (системно, изменить нельзя). Списание раз в минуту за следующую. Продление автоматическое каждую минуту. Завершается при нулевом балансе. Не доступен для онлайн-бронирования (неизвестно время окончания). Купить можно только при отсутствии других активных тарифов; до 99 единиц в чеке.

### 1.4 SUBSCRIPTION — Абонемент (Starter/Business)
Сохраняет время. Срок жизни задаётся отдельно — фиксированному тарифу добавляется TTL, клиент не теряет остаток при выходе. Старт срока жизни — выбор менеджера: с момента покупки / с автопродления / при логине в шелл. Автопродлевается абонементом, если их два (один активный). Можно покупать несколько абонементов. Если задан график действия — тарифная сессия привязывается к нему.

### Поля тарифа (общие)
Название (2–24 симв.), состояние (Активный/Неактивный), длительность, срок жизни, цена по группам ПК (per-zone, до 999 999, дробные через «.»), штрихкод. **График действия** — период активности (например, Пн–Пт 08:00–12:00). **Время продажи** — когда можно купить (например, Пн/Ср/Пт 10:00–18:00). Флаги: применение скидок клуба/клиента, виден в шелле, виден оператору, выделить в шелле, использовать в праздник, Telegram-оповещение, доступен для онлайн-брони (только FIXED/PACKAGE).

### Сравнение типов
| Параметр | FIXED | PACKAGE | PER_MINUTE | SUBSCRIPTION |
|---|---|---|---|---|
| Длительность | задаётся | не задаётся | 1 мин (системно) | задаётся |
| Срок жизни | — | — | — | задаётся |
| Старт | с покупки | с покупки | с покупки/возобновления | покупка/автопродление/логин |
| Продление | FIXED/PACKAGE, можно автопродление PER_MINUTE | то же | автоматически каждую минуту | абонементом автоматически |
| Онлайн-бронь | да | да | нет | нет |
| Сохранение времени при выходе | до конца | да (остаток до завершения) | — (баланс на депозит) | да (до срока жизни) |

Прочее: сортировка drag-and-drop, удаление через корзину, выделение тарифа («Выделить в шелле» — Starter/Business), пересоздание при апдейте (старая версия в архиве для аналитики).

## 2. Магазин (POS — трёхколоночный)

URL: `/shop`. Доступ — только при открытой кассовой смене сотрудника. Алгоритм продажи: ввести логин клиента → выбрать товар/услугу/комбо (повторный клик увеличивает количество) → указать скидку/промокод (скидка работает, если в карточке включена применимость) → выбрать способ оплаты. **Внесение на склад**: кнопка справа сверху → количество штук → опциональный комментарий.

## 3. Товары, услуги и комбо-наборы

### Товар
Поля: изображение, название (2–50 Unicode), цена закупки и цена продажи (0–999 999, дробные до 3 знаков), штрихкод (13 симв., до 10 шт.), Честный знак (требует сканирования Data Matrix при продаже/возврате), подакцизный (отметка в чеке, виден при включённой кассовой интеграции), налог по умолчанию, применимость скидки, группа товаров, отображение в шелле, Telegram-уведомление об остатках, описание, выделить в шелле (домашняя страница). Колонки списка: остаток на складе, цена закупки, цена продажи, стоимость остатков. Контекстное меню (ПКМ): оприходовать (Внесение/Списание + комментарий → лог), действия с товарами (история операций, CSV-экспорт), редактировать, отображать в шелле, добавить/убрать из группы. Товары в комбо-наборах редактировать нельзя.

### Группы товаров
Поля: название (2–50), график действия (дни недели + время доступности продажи и показа в шелле; можно оставить пустым), отображать в шелле. Добавление: «Добавить группу» в правом верхнем углу.

### Услуга
Поля: название (2–50), цена (0–999 999, дробные до 3 знаков), применимость скидки, штрихкод (13 симв.). Реализация — через `/shop` сотрудником с открытой сменой; менеджер не может продать со своего интерфейса (кроме своей открытой смены).

### Комбо-набор
Поля: название (2–50), группа ПК (если есть тариф — обязательно выбрать зал), состав (≥2 элемента): можно добавить **один** тариф (FIXED или PACKAGE, цену снижать нельзя), несколько товаров (цену в наборе можно снижать; кол-во наборов = min остаток), несколько услуг (цену можно снижать), применимость персональной скидки. Базовая цена показывается отдельно от цены продажи. Недоступен к продаже если: тариф недоступен / товар на остатке < требуемого.

## 4. Подакцизный товар
Согласно 54-ФЗ касса должна передавать «Признак предмета расчёта» = «ПОДАКЦИЗНЫЙ ТОВАР»/«АТ». Касса должна быть зарегистрирована с параметром подакциза (иначе — перерегистрация). Включение: Товары и услуги → Товары → карточка → раздел «Дополнительно» → тумблер «Подакцизный товар» (виден только при включённой кассовой интеграции).

## 5. Штрихкод
Графический код для автоматизации учёта. Добавляется в карточке товара (до 10 на товар), услуги (1), тарифа (1) — сканированием или вручную. Использование: оприходование оператором (модалка прихода → поле поиска → количество → комментарий, например, номер накладной); мгновенное добавление при продаже (вкладка Магазин или выбор тарифа на карте/таблице → поиск → товар попадает в корзину → количество).

## 6. Честный знак
Маркировка ЦРПТ. Подготовка: 1) УКЭП в УЦ ФНС России (с 01.01.2022); 2) СКЗИ/драйверы токенов; 3) регистрация на `markirovka.crpt.ru/register`, добавить товарную группу в Профиле, указать тип участника (Производитель/Импортёр/Опт/Розница), подписать пакет с Оператором ЦРПТ; 4) опц. обновление прошивки ККТ (54-ФЗ); 5) обновить ПО товарного учёта; 6) 2D-сканер штрихкода; 7) ЭДО. Включение: Товары → карточка → тумблер «Честный знак». Продажа: режим Оператора → `/shop` → товар → сканировать Data Matrix с упаковки (CR/LF в сканере включены) → способ оплаты → завершить. Товары с маркировкой не суммируются — каждый отдельной строкой со своим кодом. Удаление из корзины — повторное сканирование. Требуется фискальный аппарат + сканер штрих-кодов.

## 7. Настройки (6 вкладок)

URL: `/settings`. Доступ из Доп. меню.

- **Панель управления** — основные параметры клуба: поминутный тариф, тарифы выходного дня, сеансы, система лояльности, рабочие смены.
- **Шелл** — настройки клиентской оболочки.
- **Безопасность** — ограничения нежелательных действий клиента во время игровой сессии.
- **Кастомизация** — внешний вид шелла: цвета, логотип клуба, оформление.
- **SmartGamer** — настройки мобильного приложения для клиентов и онлайн-бронирования.
- **Интеграции** — модули: Telegram (уведомления + выбор типов событий), кассовый модуль (название пополнений депозитов, кассир, СНО категорий продаж), CloudPayments (онлайн-платежи), СБП (пополнение депозита из шелла/мобильного, РФ), Kaspi QR (пополнение депозита, Казахстан).
