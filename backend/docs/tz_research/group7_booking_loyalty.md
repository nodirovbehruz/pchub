# Группа 7: Бронирование, лояльность, отзывы

Источник: 5 HTML-страниц SmartShell support (Бронирование, Онлайн-бронирование, Система лояльности, Отзывы и вызов администратора, Отзывы клиентов).

## 1. Бронирование (модель Booking)

Раздел `admin.smartshell.gg/bookings`. Доступен в подписках Starter и Business. График: хосты по вертикали, время по горизонтали (активный диапазон 36 часов); брони, пересекающие 00:00, отображаются фиолетовым. На странице — выбор даты, сортировка групп ПК, кнопка «Забронировать».

Поля модели Booking:
- Клиент: номер телефона уже зарегистрированного пользователя или пусто (для гостя), плюс опциональное поле «дополнительная информация» (имя/телефон).
- ПК[]: массив выбранных ПК (одна бронь может включать несколько мест).
- От: дата и время начала.
- До: предполагаемое время сеанса (расчётное «до»).
- Статус: «Активные» (время начала ещё не подошло) и «Завершённые» (время окончания истекло либо завершена оператором).

Правила: бронь видна клиенту за 12 часов до начала; нельзя установить сессию, пересекающую бронь; поминутный и фиксированный тарифы автоматически завершаются к моменту брони. В шелле клиент видит ближайшую бронь и не может купить тариф, превышающий её начало.

## 2. ПКМ-меню создания брони

ПКМ на нужном ПК на карте/таблице → «Забронировать» (альтернатива — быстрые действия оператора на дашборде или кнопка «Забронировать» на вкладке «Бронирование»). Шаги: ввести телефон (опц.), указать дату/время начала и предположительное время сеанса, выбрать количество ПК, доп. информацию, нажать «Забронировать». При бронировании из карты время автоматически переносится на ближайшее свободное после текущего сеанса/брони + интервал «жёсткого бронирования». Изменение: ПКМ → «Список бронирования» → карандаш → «Обновить». Удаление: тот же путь → иконка «Стоп».

## 3. Онлайн-бронирование (настройки)

Включается в настройках Панели управления (вкладка настроек). Можно запретить онлайн-бронь отдельных ПК через ПКМ → «Управление ПК» → «Запретить онлайн бронирование». Клиент бронирует через `gamer.smartshell.gg` или мобильное приложение SmartGamer: выбирает клуб (только те, где он уже был; клубы с разрешённым онлайн-бронированием помечены), затем дату/время, тариф и игровое место. Если включено онлайн-пополнение депозита и средств недостаточно, ЛК предложит пополнить. В настройках клуба настраивается **штраф за позднюю отмену бронирования** (мин/макс время бронирования и горизонт отмены задаются в общих настройках Шелл/онлайн-брони).

## 4. Система лояльности (полностью)

Отдельный раздел панели. Четыре подсистемы:

### Скидки (`/discounts`)
Снижение стоимости тарифа/товара/услуги/комбо. Поля при создании: название; значение в процентах; флаг уведомлений в Telegram; график действия (расписание); включена/выключена. Применяется только к категориям, где явно включено применение.

### Промокоды (`/promo-codes`)
Используются для маркетинга. Поля: Название; Тип вознаграждения (скидка / пополнение депозита фиатом / пополнение бонусами); Значение (процент для скидки, сумма для пополнения); Клиенты (конкретные/группа/все); Количество использований (максимум применений); Позиции (тарифы/товары/услуги/комбо; пусто = любые); Области применения (панель управления; мобильное приложение — скоро; Shell — скоро); Активен (переключатель); Период действия (пусто = бессрочный); Уведомления в Telegram о применении и окончании. Лимит: один клиент = один раз каждый промокод.

### Кешбэк (`/cashback`)
Бонус за пополнение депозита. Поля условия: сумма пополнения (порог); тип начисления (процент или фиксированное значение); размер; срок действия (можно отключить ползунком для бессрочности). Лимиты: нельзя создать два условия с одинаковой суммой пополнения; промокоды не учитываются при расчёте кешбэка; при возврате пополнения кешбэк списывается.

### Достижения (`/achievements`)
Поля: название; условие (триггер — регистрация, пополнение баланса, траты, часы в клубе и т.п.); награда (тип); значение (размер награды, если применимо). Иконка задаётся при создании достижения вместе с названием. Правила: награду можно получить только один раз; достижение нельзя редактировать (только пересоздать); подсчёт условий начинается с момента создания.

## 5. Вызов администратора из шелла

Клиент (авторизованный или нет) нажимает кнопку «Позвать админа» в шелле — оператору смены приходит уведомление. Повторный запрос возможен через 1 минуту (rate-limit).

## 6. Отзывы клиентов (рейтинг и модерация)

После окончания сеанса в шелле появляется модальное окно: оценка 1–5, комментарий, опциональные контакты, опциональные чаевые (списываются с депозита), кнопка «Отправить оценку». Возможен анонимный отзыв. Включение: Настройки → Шелл → ползунок; после включения шелл перезапустить. Страница «Отзывы клиентов» в панели: среднее значение и сумма по каждой оценке, поиск/сортировка, список с полями — никнейм (или «гость»), оценка, название игрового места, дата/время, развёртываемый текст, чаевые. Статусы модерации: «Не прочитано» (значок 🔵) → после клика «Прочитано» (две галочки). Опционально — уведомления в Telegram-канал о новых отзывах.
